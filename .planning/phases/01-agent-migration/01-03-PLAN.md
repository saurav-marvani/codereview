    Template path (Sandbox.create(templateId, opts)):
    ```typescript
    const sandbox = await Sandbox.create(templateId, {
        timeoutMs: SANDBOX_TIMEOUT_MS,
        apiKey,
        metadata,
        lifecycle: { onTimeout: 'pause', autoResume: true },  // SBX-03: pause not kill; autoResume must be explicit
    });
    ```

    No-template path (Sandbox.create(opts)):
    ```typescript
    const sandbox = await Sandbox.create({
        timeoutMs: SANDBOX_TIMEOUT_MS,
        apiKey,
        metadata,
        lifecycle: { onTimeout: 'pause', autoResume: true },  // SBX-03: pause not kill; autoResume must be explicit
    });
    ```

    Both overloads accept lifecycle?: SandboxLifecycle per SDK type at node_modules/e2b/dist/index.d.ts:6590.

    Add two new public methods to E2BSandboxService:

    async pauseAfterIdle(sandboxId: string, idleMs: number): Promise<void>
    - Gets the apiKey from configService.get<string>('API_E2B_KEY')
    - Calls await Sandbox.setTimeout(sandboxId, idleMs, { apiKey })
    - Purpose: called by SandboxLeaseManager.release() when leaseCount hits 0 to set 5-min idle window before pause

    async connectExisting(sandboxId: string): Promise<Sandbox>
    - Gets apiKey from configService
    - Returns await Sandbox.connect(sandboxId, { apiKey })
    - SDK doc: "If the sandbox is paused, it will be automatically resumed" (node_modules/e2b/dist/index.d.ts:7990)
    - Purpose: called by SandboxLeaseManager.acquire() when state === 'READY' to reconnect

    Do NOT modify the createSandboxWithRepo() public signature — callers are unchanged.
    Do NOT change SANDBOX_TIMEOUT_MS (45 min ceiling at create time is correct; idle timeout is set separately by release()).
    Use createLogger from @kodus/flow for any new logging (CONVENTIONS.md).

    The RESEARCH.md Common Pitfall 1 is the primary guard: autoResume MUST be true — not just onTimeout: 'pause'. Add a comment above each Sandbox.create() call site: "// autoResume: true is required — SDK default is false; omitting it causes Sandbox.connect() to throw on paused sandboxes".
