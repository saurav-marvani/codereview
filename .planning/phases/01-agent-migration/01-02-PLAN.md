  <action>
    Create libs/sandbox/infrastructure/repositories/schemas/sandbox-lease.model.ts. Mirror libs/core/infrastructure/metrics/schemas/metrics-event.schema.ts for the @Schema/@Prop/SchemaFactory pattern:

    ```typescript
    @Schema({ collection: 'sandbox_leases', timestamps: false })
    export class SandboxLeaseModel extends Document {
      @Prop({ type: String, required: true })
      _id: string;            // prKey: "{orgId}:{repoId}:{prNumber}"

      @Prop({ type: String, required: false })
      sandboxId?: string;     // E2B sandbox ID; null while state === 'CREATING'

      // 'INVALIDATED' is required: the mid-create invalidation race (RESEARCH.md Pitfall 5)
      // needs to mark a lease as invalidated while create is still in-flight.
      @Prop({ type: String, required: true, enum: ['CREATING', 'READY', 'PAUSED', 'INVALIDATED'] })
      state: string;

      @Prop({ type: Number, required: true, default: 0 })
      leaseCount: number;     // ref-count of active leases

      @Prop({ type: Date, required: true })
      createdAt: Date;

      @Prop({ type: Date, required: true })
      expiresAt: Date;        // used by reaper TTL query
    }

    export const SandboxLeaseSchema = SchemaFactory.createForClass(SandboxLeaseModel);
    SandboxLeaseSchema.index({ expiresAt: 1 });                       // reaper range scan
    SandboxLeaseSchema.index({ sandboxId: 1 }, { sparse: true });     // invalidate-by-sandboxId
    ```

    Do NOT use MongoDB TTL index (expireAfterSeconds) — the reaper must kill E2B before deleting the doc.

    Create libs/sandbox/infrastructure/repositories/sandbox-lease.repository.ts as @Injectable() class. Use @InjectModel(SandboxLeaseModel.name) to inject the model. Implement these methods:

    upsertAcquire(prKey: string, leaseTtlMs: number): Promise<SandboxLeaseModel>
    - Single findOneAndUpdate call with BOTH operators in one update document:
      filter: { _id: prKey }
      update: {
        $setOnInsert: { state: 'CREATING', createdAt: now, expiresAt: now+leaseTtlMs },
        $inc: { leaseCount: 1 },
      }
      options: { upsert: true, new: true }
    - MongoDB applies $setOnInsert only on the insert path (sets leaseCount to the $inc'd value
      of 1); on subsequent updates $setOnInsert is a no-op but $inc still increments leaseCount.
    - After the call: doc.leaseCount === 1 means this caller is the creator (new doc inserted);
      doc.leaseCount > 1 means the doc already existed and this caller must poll for READY state.
    - CRITICAL: Do NOT split into find + update — atomicity required (see RESEARCH.md Pitfall 2).
    - Do NOT add a separate incrementLease() method — upsertAcquire handles both insert and
      concurrent-acquire in one atomic call.

    decrementLease(prKey: string): Promise<SandboxLeaseModel | null>
    - findOneAndUpdate filter { _id: prKey }, update { $inc: { leaseCount: -1 } }, { new: true }

    updateReady(prKey: string, sandboxId: string): Promise<void>
    - updateOne filter { _id: prKey, state: 'CREATING' }, update { $set: { state: 'READY', sandboxId } }

    findByPrKey(prKey: string): Promise<SandboxLeaseModel | null>
    - findOne({ _id: prKey })

    findExpired(now: Date): Promise<SandboxLeaseModel[]>
    - find({ expiresAt: { $lt: now } }) — reaper uses this; matches crashed-worker leases regardless of leaseCount

    delete(prKey: string): Promise<void>
    - deleteOne({ _id: prKey })
  </action>

    Create libs/sandbox/infrastructure/services/sandbox-lease-manager.service.ts as @Injectable() class implementing ISandboxLeaseManager.

    Constructor injects:
    - @Inject(SANDBOX_PROVIDER_TOKEN) private readonly sandboxProvider: ISandboxProvider
    - private readonly leaseRepo: SandboxLeaseRepository
    - private readonly configService: ConfigService (for API_E2B_KEY used in soft-drain)

    IDLE_TIMEOUT_MS constant = 300_000 (5 minutes). DEFAULT_LEASE_TTL_MS = 30 * 60 * 1000 (30 minutes).

    acquire(prKey: string, consumer: string, leaseTtlMs = DEFAULT_LEASE_TTL_MS): Promise<AcquireResult>
    1. Call leaseRepo.upsertAcquire(prKey, leaseTtlMs) — atomic insert-or-inc
    2. If doc.leaseCount === 1 (we are the creator):
       a. call sandboxProvider.createSandboxWithRepo(params from stored context — NOTE: in this plan, the acquire signature does not yet carry clone params; add a second optional argument cloneParams?: CreateSandboxParams for the full pipeline integration in plan 01-04; for now construct a minimal stub that satisfies the type)
       b. call leaseRepo.updateReady(prKey, sandbox.sandboxId or empty string for non-E2B providers)
       c. return { sandbox, leaseId: generateLeaseId(), sandboxId: ... }
    3. If doc.leaseCount > 1 and doc.state === 'CREATING': poll leaseRepo.findByPrKey every 500ms, max 30s. When state === 'READY': call Sandbox.connect(doc.sandboxId, { apiKey: configService.get('API_E2B_KEY') }) and return AcquireResult. If 30s elapsed without READY, throw SandboxCreateTimeoutError.
    4. If doc.state === 'READY' and doc.sandboxId: connect immediately (same connect path as step 3).
    5. If doc.state === 'INVALIDATED': throw with message 'sandbox invalidated for this PR'.

    release(leaseId: string): Promise<void>
    - This plan's implementation: decrementLease by prKey. Since leaseId is a UUID, store a Map<leaseId, prKey> in memory in this service (acceptable for single-worker phase 1). When leaseCount reaches 0 after decrement: if provider is E2B and sandboxId exists, call Sandbox.setTimeout(sandboxId, IDLE_TIMEOUT_MS, { apiKey }). Do NOT call sandbox.kill().

    invalidate(prKey: string): Promise<void>
    - Find lease doc by prKey. If state === 'CREATING': set state = 'INVALIDATED', do not kill yet — the create path will check and kill after completing.
    - If state === 'READY' or 'PAUSED': call Sandbox.setTimeout(sandboxId, 60_000, { apiKey }) for soft-drain (60s window for in-flight tool calls to finish), then delete the Mongo doc. Log the invalidation with prKey.
    - If doc not found: no-op (idempotent).

    generateLeaseId(): string — uses crypto.randomUUID() or uuid.v4().

    SandboxCreateTimeoutError: simple class extending Error, exported.

    Update libs/sandbox/modules/sandbox.module.ts:
    - Add MongooseModule.forFeature([{ name: SandboxLeaseModel.name, schema: SandboxLeaseSchema }]) to imports

    - Add SANDBOX_LEASE_MANAGER_TOKEN to exports

    Logging: use createLogger from @kodus/flow in the service (CONVENTIONS.md pattern). Log acquire/release/invalidate events with prKey and consumer fields.
