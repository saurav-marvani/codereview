    ```typescript
    const prKey = `${context.organizationAndTeamData.organizationId}:${context.repository.id}:${context.pullRequest.number}`;
    const { sandbox, leaseId } = await this.leaseManager.acquire(prKey, 'review');

    ```

    The SandboxInstance.cleanup function must point to release(), not kill(). Set it after acquiring:
    ```typescript
    sandbox.cleanup = async () => {
        await this.leaseManager.release(leaseId);
    };
    ```

    Handle the retry block (current lines ~201–213 in the file): these lines call
    `this.sandboxProvider.createSandboxWithRepo(...)`. After removing sandboxProvider, this block
    must NOT reference sandboxProvider. Replace it with a second acquire call using option (a)
    from B3 — acquire has "create or reuse" semantics, calling it again is the correct retry:

    ```typescript
    // Retry path: release any partial state, then re-acquire
    const retryResult = await this.leaseManager.acquire(prKey, 'review');
    cleanup = async () => {
        await this.leaseManager.release(retryResult.leaseId);
    };
    return this.updateContext(context, (draft) => {
        draft.sandboxHandle = retryResult.sandbox;
        retryResult.sandbox.cleanup = cleanup;
    });
    ```

    The cloneInfoRetry params that were passed to createSandboxWithRepo in the old retry block can
    be discarded — the lease manager owns clone params via the provider it constructed. SandboxLeaseManager
    errors propagate naturally to the outer catch block (self-contained mode fallback).

    Do NOT modify CodeReviewPipelineObserver — it calls context.sandboxHandle.cleanup() which now
    points to release(). No change needed there.

    Update libs/code-review/pipeline/code-review-pipeline.module.ts:

    - Ensure SANDBOX_LEASE_MANAGER_TOKEN is available to CreateSandboxStage via the module's provider graph
