<objective>
Reduce the remaining Phase 01 verification gap by adding best-effort force-push invalidation for GitHub synchronize events only.

Purpose: GitHub does not expose a first-class force-push flag on `pull_request.synchronize`, but the webhook payload and current PR commit list provide enough information for a narrow heuristic: if the old head SHA no longer exists in the PR commit list after synchronize, treat it as a rewritten branch and invalidate the sandbox.

Non-goal: This plan does not extend the behavior to Bitbucket, Azure, or Forgejo, and it does not promise full criterion 4 closure on its own.

Output: targeted GitHub handler change, regression tests, and refreshed verification notes.
</objective>

<task type="auto">
  <name>Task 2: Implement best-effort GitHub synchronize detection without widening platform interfaces</name>
  <files>
    libs/platform/infrastructure/webhooks/github/githubPullRequest.handler.ts
  </files>
  <action>
    Modify `GitHubPullRequestHandler` to add a small internal helper or inline block dedicated to synchronize invalidation.

    Constraints:
    - Use the existing `CodeManagementService.getCommitsForPullRequestForCodeReview()` method. Do NOT add a new cross-platform interface method just for GitHub.
    - Keep the logic GitHub-local to avoid fake abstraction.
    - Fail open: if the commit lookup fails or payload lacks the necessary SHAs, log and skip the invalidation check.

    Recommended algorithm:

    ```typescript
    if (payload?.action === 'synchronize') {
        const previousHeadSha = payload?.before;
        const currentHeadSha = payload?.after || payload?.pull_request?.head?.sha;

        if (
            previousHeadSha &&
            currentHeadSha &&
            previousHeadSha !== currentHeadSha
        ) {
            try {
                const commits =
                    await this.codeManagement.getCommitsForPullRequestForCodeReview({
                        organizationAndTeamData: context.organizationAndTeamData,
                        repository: {
                            id: repository.id,
                            name: repository.name,
                        },
                        prNumber: payload?.pull_request?.number,
                    });

                const stillContainsPreviousHead = (commits ?? []).some(
                    (commit) => commit?.sha === previousHeadSha,
                );

                if (!stillContainsPreviousHead) {
                    const prKey = `${context.organizationAndTeamData.organizationId}:${repository.id}:${payload?.pull_request?.number}`;
                    await this.outboxRepository.create({
                        jobId: undefined,
                        exchange: 'sandbox.events',
                        routingKey: SANDBOX_INVALIDATE_ROUTING_KEY,
                        payload: {
                            prKey,
                            reason: 'force_pushed',
                        } satisfies SandboxInvalidatePayload,
                    });
                }
            } catch (err) {
                this.logger.warn({
                    message: '[SBX-05] Failed to evaluate GitHub force-push heuristic',
                    context: GitHubPullRequestHandler.name,
                    error: err instanceof Error ? err : undefined,
                    metadata: {
                        repositoryId: repository.id,
                        prNumber: payload?.pull_request?.number,
                        previousHeadSha,
                        currentHeadSha,
                    },
                });
            }
        }
    }
    ```

    Place the new logic next to the existing synchronize flow so the behavior remains easy to audit.

    Important:
    - Do not remove the existing TODO comments for other platforms.
    - Do not change the closed-path invalidation logic.
    - Do not rely on git clone state or direct shell git commands in the webhook path.
  </action>

    Expected: PASS
  </verify>
