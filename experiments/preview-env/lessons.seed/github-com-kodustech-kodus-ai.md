- Boot the whole stack with 'pnpm run setup' (generates .env from .env.schema) then 'docker compose -f docker-compose.dev.yml up -d db_postgres db_mongodb rabbitmq kodus-api webhooks worker' (or 'pnpm run docker:up'). The Dockerfile copies packages/kodus-common and kodus-flow into node_modules itself, so @kodus/kodus-common does NOT need a private registry token. The app comes up HEALTHY on the auto-generated .env WITHOUT any real third-party keys — API /health on port 3001 returns 200, webhooks on 3332. Only features needing real LLM/GitHub-App keys are degraded; do not treat missing third-party creds as a boot blocker.
- Required third-party creds (list as requiredEnv, boot works without them): API_OPEN_AI_API_KEY, API_ANTHROPIC_API_KEY, API_GOOGLE_AI_API_KEY, API_MORPHLLM_API_KEY, API_EXA_KEY, API_GITHUB_APP_ID/CLIENT_SECRET/PRIVATE_KEY, API_RESEND_API_KEY. The docker build takes ~10 min (builds api/worker/web/webhooks/mcp-manager); use generous timeouts and run it backgrounded, then poll 'docker ps' for healthy.
- Jest: use 'node ./node_modules/.bin/jest --config jest.config.ts --testPathPatterns=<regex>' (PLURAL testPathPatterns). Do NOT use 'pnpm run test -- --testPathPattern=...' — the wrapper adds a stray '--' and the singular --testPathPattern is deprecated, so jest treats the whole thing as a literal filename and matches 0 tests. Correct form runs fine: 'libs/core' => 17 suites / 108 tests pass in ~6s.
- Health check: the only endpoint is GET /health on port 3001 (there is NO /api/health/deep or /health/deep — both 404). It already includes DB status; assert with: curl -fsS http://localhost:3001/health | jq -e '.details.database.postgres.status == "up" and .details.database.mongodb.status == "up"'. Webhooks health is GET :3332/health.
